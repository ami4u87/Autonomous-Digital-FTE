## Financial Rules
- Never take financial actions over $100 without approval
- For payments: always create a file in `Pending_Approval/` before processing
- All financial transactions must be logged in `Accounting/`

## Urgency Protocol
- Flag urgent keywords: **urgent**, **asap**, **invoice**, **payment**, **help**
- Urgent items are escalated immediately and placed in `Needs_Action/`

## File Management
- Never delete files, only move to `Done/` or `Rejected/`
- All task files must include a created date and status
- Completed tasks are archived in `Done/` with a completion timestamp

## Decision Logging
- Log every decision in `Logs/`
- Include: date, context, decision made, and reasoning
- Financial decisions require approval trail documentation

## Approval Workflow
1. Task arrives → `Needs_Action/`
2. Work begins → `In_Progress/`
3. If approval needed → `Pending_Approval/`
4. CEO approves → `Approved/` → execute
5. CEO rejects → `Rejected/` with reason noted
6. Task complete → `Done/`
